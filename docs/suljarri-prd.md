## 1. Executive Summary

술자리에서 대표자(총무)가 방을 만들고 QR/링크를 공유하면, 참여자들이 소주·맥주를 원클릭으로 기록하고, 종료 후 주종별 총액을 입력하면 마신 잔 비율로 인당 정산액이 자동 계산되는 초경량 웹앱이다. 설치·로그인 없이 링크 하나로 시작하며, "마신 만큼 공정하게 내자"는 한국 술자리 문화의 오랜 페인포인트를 해결한다. MVP는 v1 핵심 기능 완성 후 광고 수익 기반으로 운영하며, v2에서 카카오 로그인 + 카카오페이 정산 연동으로 확장한다.

## 2. Background & Context

### 문제 공간

- 한국 술자리에서 정산은 대부분 n/1 (인원수 나누기)로 처리되어 "많이 마신 사람이 이득"이라는 구조적 불공정이 존재
- 기존 더치페이 앱(더치트 등)은 음료 종류별 잔 수 기록이 아닌 금액 입력 방식이라 술자리 맥락에 맞지 않음
- 앱 설치 또는 회원가입이 필요한 앱은 "오늘 처음 쓰는 사람"에게 채택 장벽이 너무 높음
- 총무는 술자리 종료 후 지친 상태에서 계산기를 두드리며 소주값/맥주값을 나누고, 카카오페이 개별 요청까지 직접 처리해야 함

### 해결 방향

- 링크/QR 하나로 진입, 이름만 입력해서 즉시 참여
- 술자리 중 원클릭으로 잔 수 기록 → 종료 시 주종별 총액 입력 → 자동 정산
- "깍두기" 옵션으로 특정 인원의 음주 기록을 정산에서 제외
- 결과를 시각적으로 보여주고 카카오톡으로 공유

## 3. Objectives & Success Metrics

### Goals

1. 술자리 총무가 정산 계산에 쓰는 시간을 10분 → 1분 이내로 단축
2. 참여자 모두가 "마신 만큼" 낸다는 공정함을 느끼게 함
3. 설치·로그인 없이 방 생성 → 참여 → 정산 전체를 10탭 이내로 완료
4. 카카오톡 공유 기반 바이럴로 별도 마케팅 없이 사용자 확보

### Non-Goals (v1 스코프 외)

1. **카카오 로그인 / 카카오페이 연동** → v2에서 처리
2. **소주·맥주 외 주종 지원** (막걸리, 칵테일 등) → v2 이후
3. **20명 이상 방** → 무료티어 한계, v3 유료 플랜에서 처리
4. **과거 정산 이력 조회** → 로그인 없는 구조에서는 불가
5. **기업 경비 처리, 세금계산서** → 타겟 외
6. **글로벌 다국어 지원** → 한국 시장 우선

### Success Metrics

| 지표                | 현재 | 목표 (출시 후 4주) | 측정 방법                      |
| ------------------- | ---- | ------------------ | ------------------------------ |
| 정산 완료율         | -    | 60% 이상           | 방 생성 수 대비 정산 완료 수   |
| 방당 평균 참여자 수 | -    | 4명 이상           | Firebase 이벤트                |
| 참여자 기록 완료율  | -    | 70% 이상           | 참여자 중 1잔 이상 기록한 비율 |
| 결과 공유 클릭률    | -    | 50% 이상           | 공유 버튼 클릭 / 정산 완료 수  |
| 주간 방 생성 수     | -    | 100개              | Firebase 이벤트                |

## 4. Target Users & Segments

### Primary: 술자리 총무

- **프로필**: 20~35세, 직장인 또는 대학생, 모임을 주도하는 사람
- **상황**: 회식, 동창 모임, 소모임 등 정기적으로 술자리를 주최
- **페인포인트**: 정산 계산 + 카카오페이 요청이 귀찮고 분쟁 소지가 있음
- **니즈**: 빠르고, 납득 가능하고, 공정한 정산

### Secondary: 술자리 참여자

- **프로필**: 총무가 공유한 링크로 접속하는 모든 참여자
- **상황**: 링크를 받아 처음 앱을 접하는 상황 (설치 없이 바로 써야 함)
- **페인포인트**: n/1 정산의 불공정함, 많이 마신 사람에게 손해 보는 느낌
- **니즈**: 설치 없이 바로 참여, 내가 마신 만큼만 냄

### Acquisition: 결과 공유를 받는 사람

- 정산 결과 이미지를 단톡방에서 보고 앱을 인지 → 다음 술자리에서 총무로 사용

## 6. Solution Overview

### 아키텍처

```
[Client: React SPA]
  ├── 방 생성 → Firestore에 고유 roomId 생성
  ├── QR코드: qrcode.js 라이브러리
  ├── 실시간 동기화: Firestore onSnapshot (잔수 합계만)
  ├── Rate Limit: 참여자당 10초에 1회 기록 제한 (클라이언트 + Firestore Rules)
  └── 광고: Google AdSense 하단 고정 배너

[Firestore 구조]
  rooms/{roomId}
    createdAt: timestamp
    closedAt: timestamp | null      ← 종료 시각 기록 (24시간 후 삭제 기준)
    hostId: string
    status: "active" | "closed"
    deleteAt: timestamp             ← closedAt + 24h, TTL 삭제 스케줄용

  rooms/{roomId}/participants/{participantId}
    name: string
    status: "active" | "left"       ← 중도 하차 여부
    isKkakdugi: boolean
    soju: number                    ← 잔수 합계 (전체 동기화 대상)
    beer: number                    ← 잔수 합계 (전체 동기화 대상)
    lastTapAt: timestamp            ← rate limit용

  rooms/{roomId}/logs/{logId}       ← 개별 클릭 로그 (전체 동기화 X, 로컬 기록용)
    participantId: string
    type: "soju" | "beer"
    action: "add" | "subtract"
    timestamp: timestamp

  rooms/{roomId}/settlement
    sojuTotal: number
    beerTotal: number
    result/{participantId}: number
```

### 동기화 전략

| 데이터                            | 동기화 방식                   | 이유                                                      |
| --------------------------------- | ----------------------------- | --------------------------------------------------------- |
| `participants/{id}.soju`, `.beer` | Firestore onSnapshot (실시간) | 리더보드에 필요한 최소 데이터만 전파                      |
| `logs/{logId}`                    | 로컬 쓰기만, 구독 없음        | 개별 클릭 이벤트를 전체에 뿌리면 동시 연결 한도 초과 위험 |
| `settlement`                      | 종료 시 1회 write             | 정산 완료 후 단발성                                       |

### 데이터 생명주기

- 술자리 종료 시 `closedAt` + `deleteAt`(closedAt + 24h) 기록
- Cloud Firestore TTL 또는 Cloud Scheduler로 `deleteAt` 지난 방 자동 삭제
- 활성 방만 유지되므로 데이터 누적 없음

### 핵심 정산 로직

```
깍두기 제외한 참여자만 계산에 포함

소주 단위 기여액 = 소주 총액 / Σ(비깍두기 참여자 소주 잔수)
맥주 단위 기여액 = 맥주 총액 / Σ(비깍두기 참여자 맥주 잔수)

개인 부담액 = ceil(소주 단위 기여액 × 개인 소주 잔수)
           + ceil(맥주 단위 기여액 × 개인 맥주 잔수)
```

### 화면 플로우

```
[홈] → [방 생성] → [방 화면 (대기/진행)]
                              ↓
                   [참여자: 링크/QR 접속]
                   [이름 입력 → 음주 기록 화면]
                              ↓
                   [대표자: 종료 버튼]
                   [소주 총액 / 맥주 총액 입력]
                              ↓
                   [정산 결과 화면]
                   [공유 / 이미지 저장]
```

### UX 원칙

- 핵심 동작(잔 수 기록)은 엄지 하나로 가능해야 함 → 버튼 크게, 하단 배치
- 하단 광고 배너는 핵심 버튼을 가리지 않도록 여백 확보
- 에러 메시지 없이 rate limit 처리: 10초 내 재탭 시 버튼 일시 비활성화 + 시각 피드백

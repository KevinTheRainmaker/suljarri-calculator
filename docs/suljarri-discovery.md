# Discovery Plan: 술자리 정산 웹앱 (가칭: 잔잔바라)

**Date**: 2026-03-18
**Product Stage**: 신규 제품
**Discovery Question**: 술자리에서 마신 양 기반 자동 정산을 로그인 없이, 앱 설치 없이 제공할 수 있는가?

---

## 제품 개요

술자리 대표자가 방을 만들어 QR/링크를 공유하면, 참여자들이 소주/맥주를 원클릭으로 기록하고, 종료 시 총액을 입력하면 마신 잔 비율로 자동 정산하는 초경량 웹앱.

### 핵심 제약

1. 다운로드 없는 웹앱
2. 로그인 없이 사용 (이름만 입력)
3. UI 매우 단순
4. 서버리스 / 로컬 중심 (Firebase 무료티어)
5. 하단 광고 배너 포함

---

## 확정된 기능 스코프 (v1)

| 기능             | 설명                                                                       |
| ---------------- | -------------------------------------------------------------------------- |
| 방 생성          | 대표자가 방 생성 → QR코드 + 링크 동시 제공                                 |
| 이름 입력 참여   | 로그인 없이 이름만 입력                                                    |
| 원클릭 음주 기록 | 소주 +1 / 맥주 +1 버튼, 10초 rate limit                                    |
| 실시간 리더보드  | 참여자별 현재 잔 수 실시간 표시                                            |
| 음주 독려 노티   | 특정 참여자가 타 멤버 대비 기록이 적으면 그룹에 "진짜 안마시고 있음?" 노티 |
| 깍두기 옵션      | 특정 참여자를 정산에서 제외 (마신 양 계산 무시)                            |
| 비율 기반 정산   | 종료 시 소주 총액 + 맥주 총액 입력 → 잔 비율로 개인별 금액 산출            |
| 시각적 요약      | 영수증 스타일 결과 화면, 카카오톡 공유 최적화                              |
| 광고 배너        | 하단 고정 배너                                                             |

### 최대 인원 제한

- Firebase 무료티어 기준으로 방당 최대 인원 제한 (예: 20명)
- 인원 초과 시 유료 전환 안내

## 기술 아키텍처 결정

```
Client (Vanilla JS / React)
  ├── Firebase Realtime DB (무료티어)
  │     ├── 방 상태 (참여자, 잔 수)
  │     └── rate limit 처리
  ├── URL/QR 기반 방 참여
  ├── 카카오 오픈그래프 메타태그
  └── 하단 광고 배너 (Google AdSense)

추후 확장:
  ├── 카카오 로그인
  └── 카카오페이 정산 딥링크
```

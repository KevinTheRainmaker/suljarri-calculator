# Product Strategy: 술자리 정산 웹앱 (가칭: 잔잔바라)

**Date**: 2026-03-18
**Stage**: Pre-MVP (아이디어 → 개발 착수 직전)

---

## 1. Vision

> "술자리에서 '나 얼마야?'라는 말이 사라지는 날까지 — 마신 만큼, 딱 그만큼만."

술자리의 마지막을 어색한 계산기 두드림이 아니라, 한 번의 탭으로 끝내는 경험을 만든다.
한국의 모든 술자리가 공정하고 유쾌하게 마무리되도록.

---

## 2. Target Segments

| 세그먼트                             | 규모    | 고통 수준 | 현재 대안                     | 우선순위    |
| ------------------------------------ | ------- | --------- | ----------------------------- | ----------- |
| 술자리 총무 (20-30대 직장인/대학생)  | 대      | 매우 높음 | 계산기 + 카카오페이 수기 요청 | Primary     |
| 술자리 참여자 (링크로 들어오는 사람) | 매우 대 | 중간      | 그냥 n/1로 냄                 | Secondary   |
| 술자리 구경꾼 (결과 공유 받는 사람)  | 중간    | 낮음      | —                             | Acquisition |

**명시적으로 타겟하지 않는 대상:**

- 기업 경비 처리 (세금계산서, 복잡한 회계 필요)
- 20명 이상 대규모 이벤트 (무료티어 한계, 복잡도 급증)
- 글로벌 시장 (초기에는 한국 술자리 문화 특화)

---

## 3. Pain Points & 창출 가치

**총무 입장:**

- 문제: 술자리 끝나고 지친 상태에서 계산기 두드리며 소주값/맥주값 나누고 카카오페이 개별 요청해야 함
- 현재 비용: 정산에 10~15분 소요, 실수로 분쟁 발생, 총무 이미지 손상
- 창출 가치: 30초 안에 정산 완료, 오해 없는 투명한 계산

**참여자 입장:**

- 문제: 나는 소주 2잔 마셨는데 맥주 5잔 마신 사람과 같은 금액을 냄
- 현재 비용: 억울하지만 말 꺼내기 어색해서 그냥 냄
- 창출 가치: 마신 만큼만 내는 공정함, 깍두기 옵션으로 명확한 예외 처리

---

## 4. Value Propositions (JTBD)

**총무에게:**

> 술자리가 끝나고 계산을 해야 할 때, 귀찮은 수작업 없이 공정한 정산을 빠르게 마무리하고 싶다.
> 그래서 모두가 납득하며 집에 갈 수 있도록.

**참여자에게:**

> 술자리에서 정산을 낼 때, 내가 마신 만큼만 정확히 내고 싶다.
> 그래서 억울함 없이 다음 술자리도 흔쾌히 참여할 수 있도록.

## 5. 전략적 Trade-offs

| 우리가 선택하는 것              | 포기하는 것             | 이유                                     |
| ------------------------------- | ----------------------- | ---------------------------------------- |
| 웹앱 (설치 없음)                | 네이티브 앱             | 마찰 제거가 바이럴의 전제조건            |
| 이름만 입력                     | 회원가입/로그인         | 첫 술자리에서 쓰려면 장벽이 0이어야 함   |
| 마신 잔 비율 정산               | 메뉴 단가 기반 정산     | 단순함이 정산 완료율을 높임              |
| Firebase 무료티어 + 인원 제한   | 무한 확장성             | 0원으로 시작해야 광고 수익이 의미 있어짐 |
| 한국 술자리 특화 UI (소주/맥주) | 다국적 음료 지원        | 깊은 문화 적합성이 얕은 범용성보다 강함  |
| 광고 배너 수익                  | 구독/프리미엄 초기 도입 | MVP 단계에서 마찰 최소화                 |

## 6. 핵심 지표

**North Star Metric:** 정산 완료 세션 수 (주간)

| 지표 유형 | 지표                | 목표 (출시 후 1개월) |
| --------- | ------------------- | -------------------- |
| Input     | 방 생성 수 (주간)   | 100개                |
| Input     | 방당 평균 참여자 수 | 4명 이상             |
| Input     | 참여자 기록 완료율  | 70% 이상             |
| Input     | 결과 공유 클릭률    | 50% 이상             |
| Health    | 정산 완료율         | 60% 이상             |
| Health    | 중간 이탈률         | 30% 이하             |
| Revenue   | AdSense RPM         | 데이터 수집 중       |

## 7. 성장 엔진

**핵심 루프: 공유 기반 바이럴**

```
총무가 방 생성
      ↓
QR/링크를 단톡방에 공유
      ↓
참여자들 링크 클릭 → 앱 첫 경험 (설치 없이)
      ↓
술자리 중 실시간 리더보드 흥미 유발
      ↓
정산 결과 이미지 → 단톡방 재공유
      ↓
구경꾼도 앱 인지 → 다음 술자리에서 총무 역할 시 사용
```

- Acquisition: 카카오톡 링크 공유 (오픈그래프 최적화)
- Activation: 첫 방 참여까지 탭 3회 이내
- Retention: 총무는 반복 사용, 참여자는 총무 경험 후 재사용
- Referral: 결과 화면 "다음 술자리에 써봐" CTA + 이미지 공유

## 9. 방어성 (Defensibility)

> 솔직하게: 초기에는 모방이 쉽다. 아래 방향으로 해자를 쌓는다.

| 단계      | 방어 요소          | 메커니즘                        |
| --------- | ------------------ | ------------------------------- |
| 초기 (v1) | 단순함 + 실행 속도 | 시장 선점, "술자리 앱" 검색 1위 |
| 중기 (v2) | 브랜드 습관화      | "술자리 = 이 앱" 전환 비용 발생 |
| 중기 (v2) | 카카오 생태계 연동 | 카카오페이 정산까지 원스톱 경험 |
| 장기 (v3) | 데이터 네트워크    | 음주 패턴 데이터 기반 개인화    |
